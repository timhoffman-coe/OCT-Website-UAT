## Securing the Browser

By late February 2026, the OCT branch's **Chrome Extension Security Pilot** had already blocked over 1,000 browser extensions as part of a transition toward a curated "allow" list for Google Chrome environments across the City.

This initiative is part of OCT's broader security modernization efforts, aimed at protecting City data from the risks posed by unvetted third-party browser extensions.

### The Problem with Unrestricted Extensions

Browser extensions can be incredibly useful, but they also represent a significant security surface. Many extensions request broad permissions — access to browsing history, the ability to read and modify data on websites, or even access to clipboard contents. Without oversight, a single compromised or malicious extension could:

- **Exfiltrate sensitive data** from internal City applications
- **Inject scripts** into web pages used for municipal business
- **Capture credentials** entered into City systems

### The Allow-List Approach

Rather than trying to identify and block individual bad actors, the security team is moving to an **allow-list model**. Under this approach:

- All extensions are blocked by default
- Extensions that have been reviewed and approved are added to the allow list
- Staff can request review of extensions they need for their work

This is a more sustainable and secure approach than maintaining a growing blocklist, and it ensures that every extension running on City-managed Chrome browsers has been evaluated for security and compliance.

### What Staff Should Know

If an extension you use regularly has been blocked, you can submit a review request through the standard IT service channels. The security team will evaluate the extension and, if it meets compliance requirements, add it to the approved list.

This pilot is an important step in protecting the City's digital infrastructure while still giving staff the tools they need to be productive.
